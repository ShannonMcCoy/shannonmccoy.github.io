1. Overview

This guide provides a concise overview of how Illinois Ambulatory Surgery Centers (ASCs) use SurgiSource to complete and submit quarterly reports. It is designed to support users responsible for data entry, validation, and submission of required quarterly state reports.

The instructions in this guide walk you through:

Accessing quarterly reporting modules

Navigating reporting views

Filtering and reviewing case data

Identifying missing or incomplete elements

Preparing data for state submission

2. Intended Audience

This document is intended for:

ASC administrators and staff responsible for completing required quarterly reporting

Clinical or operational users who enter, verify, or submit case-level data

Compliance staff responsible for reviewing reporting completeness

3. Purpose of Quarterly Reporting

Illinois ASCs are required to submit quarterly surgical case data to maintain compliance with state reporting statutes. SurgiSource provides standardized workflows to:

Capture case-level clinical and operational data

Validate missing or incomplete submissions

Generate structured quarterly reports

Monitor reporting accuracy and completion status

This guide focuses on the workflow used to review, validate, and submit required quarterly data.

4. Navigating to Quarterly Reports

To access quarterly ASC reports:

Log in to SurgiSource using your assigned credentials.

Select ASC Quarterly Reports from the main navigation menu.

Choose the reporting quarter (Q1, Q2, Q3, or Q4).

Open the reporting dashboard for detailed case review.

A typical landing page will display filters, counts of completed cases, and indicators for incomplete or missing data.

5. Reporting Dashboard Overview

The Quarterly Reporting dashboard provides:

Case counts for the selected quarter

Completion indicators (complete, incomplete, missing fields)

Filtering options to refine data sets

Navigation controls for drilling down into case-level details

Export tools for generating CSV/PDF report files

Each column provides sortable values so you can quickly locate missing data or items requiring review.
