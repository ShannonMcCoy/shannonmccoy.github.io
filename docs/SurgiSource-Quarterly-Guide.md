## 1. Overview

This guide provides a concise overview of how Illinois Ambulatory Surgery Centers (ASCs) use **SurgiSource** to complete and submit quarterly reports. It is designed to support users responsible for data entry, validation, and submission of required quarterly state reports.

This guide covers:

- Accessing quarterly reporting modules  
- Navigating reporting views  
- Filtering and reviewing case data  
- Identifying missing or incomplete elements  
- Preparing data for state submission  

---

## 2. Intended Audience

This document is intended for:

- ASC administrators responsible for quarterly reporting  
- Clinical or operational users who enter, verify, or submit case data  
- Compliance staff reviewing reporting completeness  

---

## 3. Purpose of Quarterly Reporting

Illinois ASCs are required to submit quarterly surgical case data to maintain compliance with state reporting requirements.

SurgiSource supports this process by enabling users to:

- Capture case-level clinical and operational data  
- Validate missing or incomplete information  
- Generate structured quarterly reports  
- Monitor reporting accuracy and completion status  

This guide focuses on the workflow used to review, validate, and submit required quarterly data.

---

## 4. Navigating to Quarterly Reports

To access quarterly ASC reports:

1. Log in to **SurgiSource** using your assigned credentials.  
2. Select **ASC Quarterly Reports** from the main navigation menu.  
3. Choose the required reporting quarter (Q1, Q2, Q3, or Q4).  
4. Open the **Reporting Dashboard** to begin your review.

A typical landing page displays filters, case counts, and indicators for incomplete or missing data.

## 2. Filtering, Searching, and Reviewing Case Data

SurgiSource provides filtering and navigation tools that help ASC staff quickly locate cases requiring review before quarterly submission. These tools simplify validation, highlight incomplete fields, and support accurate reporting.

---

## 2.1 Filter Panel Overview

The **Filter Panel** appears at the top of the Quarterly Reporting Dashboard. It allows users to refine case lists by:

- Date range  
- Procedure type  
- Surgeon  
- Completion status (complete/incomplete)  
- Case type  
- Facility location (if applicable)  

**Screenshot placeholder:**  
`![Screenshot: Quarterly Report Filter Panel](images/filter-panel.png)`

---

## 2.2 Applying Filters

To apply filters:

1. Open the **Filter Panel**.  
2. Select one or more criteria from the available dropdown menus.  
3. Click **Apply Filters**.  
4. The case list refreshes automatically.  

You may apply multiple filters at once to narrow the datasetâ€”for example, to isolate:

- A single surgeon  
- A specific date range  
- A particular procedure type  

---

## 2.3 Clearing Filters

To clear all filters:

1. Open the **Filter Panel**.  
2. Click **Clear All Filters**.  
3. The view resets to display all cases for the quarter.

**Screenshot placeholder:**  
`![Screenshot: Clear Filters Button](images/clear-filters.png)`

---

## 2.4 Searching for Specific Cases

Use the **Search** bar to find cases by:

- Patient name  
- Case number  
- Surgeon  
- Procedure code  

Search supports **partial match keywords**.  
For example:  
Typing `23` may return cases *2311*, *2330*, or *2398*.

---

## 2.5 Sorting Case Columns

All case list columns are sortable.  
Click any column header to sort by:

- **Date of Service**  
- **Patient Identifier**  
- **Procedure**  
- **Surgeon**  
- **Status** (Complete / Incomplete)  

Sorting helps quickly identify:

- The oldest unsubmitted cases  
- Surgeons with missing data  
- High-volume procedure types  
- Clusters of incomplete cases
